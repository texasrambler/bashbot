1. Describe project 

My chatbot project is RAG implementation using  a local Ollama model.

My goal was to create an implementation that only returns results based on
the context provided by  a local vector database.

On the right is a crude diagram of the design. There are 2 pipelines that are
utilized. The first is an ingestion pipeline that reads input from files located
in the data directory and performs the cleaning, tokenization and embedding.

These are stored as chunked documents in the vector database (currently a chromadb
implementation backed by a sqllite file).

The operational pipeline (the middle and right portions)
then prompts a user for a question, performs a similarity
search based on a threshold provided at runtime, or the default value of 0.6.

The results are then returned  (currently the max number of results is 10).

If there are no similar documents found in the vector database, then no LLM prompt
is generated and no search performed.

If documents are returned, thay are concatenated into a context that is placed into
a prompt and sent to the LLM and the response is diplayed.
