locale - get locale-specific information The command displays information
about the current locale, or all locales, on standard output.  When invoked
without arguments, displays the current locale settings for each locale
category (see based on the settings of the environment variables that control
the locale (see Values for variables set in the environment are printed
without double quotes, implied values are printed with double quotes.  If
either the or the option (or one of their long-format equivalents) is
specified, the behavior is as follows: Display a list of all available
locales.  The option causes the metadata about each locale to be included in
the output.  Display the available charmaps (character set description files).
To display the current character set for the locale, use locale -c charmap.
The command can also be provided with one or more arguments, which are the
names of locale keywords (for example, or or locale categories (for example,
or For each argument, the following is displayed: For a locale keyword, the
value of that keyword to be displayed.	For a locale category, the values of
all keywords in that category are displayed.  When arguments are supplied, the
following options are meaningful: For a category name argument, write the name
of the locale category on a separate line preceding the list of keyword values
for that category.  For a keyword name argument, write the name of the locale
category for this keyword on a separate line preceding the keyword value.
This option improves readability when multiple name arguments are specified.
