setcap - set file capabilities setcap [-q] [-n <rootuid>] [-v]
{capabilities|-|-r} filename [ ... capabilitiesN fileN ] In the absence of the
(verify) option sets the capabilities of each specified to the specified.  The
optional argument can be used to set the file capability for use only in a
user namespace with this root user ID owner. The option is used to verify that
the specified capabilities are currently associated with the file. If -v and
-n are supplied, the argument is also verified.	 The Set are specified in the
form described in The special capability string, can be used to indicate that
capabilities are read from the standard input. In such cases, the capability
set is terminated with a blank line.  The special capability string, is used
to remove a capability set from a file. Note, setting an empty capability set
is as removing it. An empty set can be used to guarantee a file is not
executed with privilege in spite of the fact that the prevailing
ambient+inheritable sets would otherwise bestow capabilities on executed
binaries.  The is used to force completion even when it is in some way
considered an invalid operation. This can affect and setting file capabilities
the kernel will not be able to make sense of.  The flag is used to make the
program less verbose in its output.  The program will exit with a 0 exit code
if successful. On failure, the exit code is 1.	Please report bugs via:
