automake - Generate Makefile.in files for configure from Makefile.am
[OPTION]... [Makefile]...  Generate Makefile.in for configure from
Makefile.am.  --help print this help, then exit --version print version
number, then exit -v, --verbose verbosely list files processed --no-force only
update Makefile.in's that are out of date -W, --warnings=CATEGORY report the
warnings falling in CATEGORY -i, --ignore-deps disable dependency tracking
code --include-deps enable dependency tracking code --foreign set strictness
to foreign --gnits set strictness to gnits --gnu set strictness to gnu -a,
--add-missing add missing standard files to package --libdir=DIR set directory
storing library files --print-libdir print directory storing library files -c,
--copy with -a, copy missing files (default is symlink) -f, --force-missing
force update of standard files cross cross compilation issues gnu GNU coding
standards (default in gnu and gnits modes) obsolete obsolete features or
constructions (default) override user redefinitions of Automake rules or
variables portability portability issues (default in gnu and gnits modes)
portability-recursive nested Make variables (default with -Wportability)
extra-portability extra portability issues related to obscure tools syntax
dubious syntactic constructs (default) unsupported unsupported or incomplete
features (default) all turn on all the warnings none turn off all the warnings
no-CATEGORY turn off warnings in CATEGORY error treat all enabled warnings as
